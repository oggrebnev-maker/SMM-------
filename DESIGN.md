## ОБЯЗАТЕЛЬНО К ИСПОЛНЕНИЮ

- Перед любой правкой верстки, стилей, классов — проверить соответствие правилам ниже
- Не вводить новые CSS-классы, переменные, размеры шрифтов без явного запроса
- Не менять существующие компоненты если задача этого не требует
- Все отступы, цвета, шрифты — только через переменные из `app.css` (`:root`)
- При малейшем сомнении — использовать существующий компонент/класс, не создавать новый
- Новые страницы и разделы верстать строго по образцу существующих аналогичных страниц

## ЗАПРЕЩЕНО

- Инлайновые стили (`style="..."`)
- Новые цвета текста кроме: `var(--text)`, `var(--text2)`, `var(--muted)`, `var(--red)`
- Новые размеры шрифтов кроме описанных в разделе «Типографика»
- Новые классы кнопок кроме: `btn-primary`, `btn-secondary`, `btn-danger`, `btn-ghost` + модификаторы размера
- Дублировать стили инлайном вместо использования `.form-group`, `.project-view-save-wrap` и других готовых классов
- Заменять `.project-view-confirm-backdrop` / `.project-view-confirm-box` на общие `.modal` без согласования

# Дизайн-правила проекта

Единый источник правды по типографике, цветам текста, кнопкам, формам и выезжающим панелям. Переменные заданы в `app.css` (`:root`). Менять значения только в `:root`; в стилях — только ссылаться на переменные.

## 1. Типографика

### 1.1. Размеры шрифта (--text-*)

| Переменная    | Значение | Назначение |
|---------------|----------|------------|
| `--text-2xs`  | 10px     | Компактные кнопки (.btn-sm), мелкие подписи |
| `--text-xs`   | 11px     | Кнопки (uppercase), бейджи, метки времени |
| `--text-sm`   | 12px     | Вторичный текст, подписи к полям, лейблы в карточках |
| `--text-sm2`  | 13px     | Мелкий основной текст, подзаголовки страниц, мета |
| `--text-base` | 14px     | **Основной текст** — параграфы, формы, списки |

| `--text-lg`   | 16px     | Заголовки модалок, заголовки карточек |
| `--text-xl`   | 18px     | Заголовки панелей и секций |

| `--font-extrabold` | 800      | Заголовки страниц, панелей, крупные акценты |

### 1.3. Роли (типовые комбинации)

## 2. Цвета текста

Использовать только эти переменные. Новые цвета текста не вводить без обоснования.

| Переменная | Назначение | Где использовать |
|------------|------------|------------------|
| **`var(--text)`** | Основной текст | Заголовки (H1, H2, H3), названия карточек, текст в полях ввода, основной контент |
| **`var(--text2)`** | Вторичный текст | Подзаголовок страницы, **лейблы полей**, подписи под блоками (dash-card-label), описание карточки, метаданные |
| **`var(--muted)`** | Приглушённый текст | Placeholder в полях, неактивные пункты, вспомогательный текст («необязательно» и т.п.) |
| **`var(--red)`** | Ошибки и опасные действия | Текст ошибки (.form-error), кнопки удаления, предупреждения |

## 3. Кнопки

**Типы:** `btn-primary` (основное действие), `btn-secondary` (Отмена, второстепенное), `btn-danger` (опасные действия), `btn-ghost` (вспомогательные). Цвета в `app.css`, блок BUTTON SYSTEM.

**Размеры (модификаторы):**
- **Размер 1** — `btn-size-1`: вертикальные отступы 20px (по умолчанию).
- **Размер 2** — `btn-size-2`: 10px (формы, интерфейс).
- **Размер 3** — `btn-size-3`: 5px (компактные кнопки).

Пример: `class="btn btn-primary btn-size-2"`. В задачах: «кнопка размера 2», «btn-primary, размер 3».

**Ширина кнопок по контексту:**

| Контекст | Поведение |
|----------|-----------|
| Футер slide-panel (`.slide-panel-footer`) | По контенту, выравнивание по правому краю |
| Карточка (`.project-view-save-wrap`) | По контенту; при необходимости на всю ширину — класс `.btn-full` |
| Футер модалки | По контенту (`.modal-footer`); на всю ширину — отдельный класс (напр. `.project-create-modal-footer`) |

**Иконки-кнопки (только иконка):** класс `.btn-icon` — 34×34px, скругление `var(--btn-radius)` (5px), фон прозрачный, без рамки, цвет `var(--text2)`; при наведении — фон `var(--surface2)`, цвет `var(--accent)`. Опасные действия (удаление): цвет `var(--red)`, при наведении — лёгкий красный фон. Стили в `app.css`, блок BUTTON SYSTEM.

## 4. Формы

### 4.1. Структура и отступы

**Форма в боковой панели** (эталон — форма UTM-меток в сайдбаре проекта / slide-panel): шапка панели — `padding: 24px 24px 16px`, граница снизу `1px solid var(--border)`; область с полями — `padding: 24px`, между блоками полей `gap: 16px`; блок кнопок — `padding: 0 24px 24px`, между кнопками `gap: 10px`.

**Формы внутри карточки (`.project-view-card`):** карточка `padding: 24px 28px 28px`; между `.form-group` — **22px**; блок «Сохранить» (`.project-view-save-wrap`) — **24px** сверху.

### 4.2. Заголовок поля и подзаголовки секций (закреплено)

- **`.form-group label`** (заголовок поля):  
  размер **`var(--text-sm)`** (12px), толщина **`var(--font-bold)`** (700), цвет **`var(--text2)`**, **uppercase**, **letter-spacing: 0.6px**. Менять только через переменные и этот раздел.
- **Подзаголовок секции внутри формы** (например, `.project-view-hashtags-list-caption`): тот же стандарт — 12px, bold, uppercase, letter-spacing 0.6px, цвет `var(--text2)`. Между секциями — **24px**.

### 4.3. Поля ввода и цвета

- **Текст в поле (input/textarea):** цвет **`var(--text)`**, шрифт 14px (`var(--text-base)`).
- **Placeholder:** цвет **`var(--muted)`**.
- **Ошибка:** цвет **`var(--red)`**, блок `.form-error`.

Поля: фон `var(--bg)`, граница `1.5px solid var(--border)`, скругление 5px, отступы 11px 14px; фокус — граница `var(--accent)`, тень `0 0 0 3px rgba(51,63,100,0.1)`. Контейнер поля — `.form-group`, зазор между label и полем **6px**.

Новые формы: использовать `.form-group`, `.project-view-save-wrap`, не дублировать стили инлайном.

## 5. Выезжающие панели (slide-panel)

Панель на весь экран, выезжает снизу. Эталон — окно настроек проекта / форма создания проекта.

**Структура:** обёртка `slide-panel` + `slide-panel-backdrop` + `slide-panel-panel` → внутри `slide-panel-header` (заголовок `slide-panel-title`, кнопка закрытия) + `slide-panel-body` (слева `slide-panel-main`, справа `slide-panel-sidebar`). При необходимости: `slide-panel-form`, `slide-panel-footer` с кнопками.

**Футер панели (`.slide-panel-footer`):** отступ сверху **8px** (на ≤767px — 12px), между кнопками **12px** (на ≤767px — 8px), выравнивание по правому краю (`justify-content: flex-end`), не сжимается (`flex-shrink: 0`). Кнопки по контенту (см. раздел 3).

**Открытие/закрытие:** класс `slide-panel--open` на обёртку; закрытие — по крестику и клику по backdrop. Кнопка закрытия: `aria-label="Закрыть"`.

**Адаптив (общие стили в app.css):** ≤1100px сайдбар 28%, ≤1000px — 30%; ≤768px — колонка (сайдбар сверху); ≤767px — уменьшенные отступы, заголовок 16px; ≤639px — отступы 12px.

Новые панели — повторять эту структуру и классы; детали и пример разметки смотреть в `#project-create-modal` / `#project-view-modal` в `assets/js/pages/projects.js`.

### 5.1. Всплывающие окна (модалки)

**Отработанное окно подтверждений и подсказок** (удаление проекта/аккаунта, подсказка «хорошо настроить проект» и т.п.): функция `projectViewConfirm()` в `projects.js`, разметка с классами **`.project-view-confirm-backdrop`** (подложка, блюр), **`.project-view-confirm-box`** (само окно), **`.project-view-confirm-title`**, **`.project-view-confirm-text`**, **`.project-view-confirm-actions`**. Стили в `app.css` (блок «Отработанное всплывающее окно»): рамка `1.5px solid var(--border)`, скругление 16px, тень `var(--shadow-lg)`, типографика заголовка и текста по п. 1.3 и 2. **Это окно не заменять** на общие `.modal` без согласования — оно единое для подтверждений и подсказок в разделе проекта.

**Прочие центрированные модалки** (формы, выбор): обёртка `.modal` → `.modal-backdrop` + `.modal-box` → `.modal-header` (заголовок `h3`), при необходимости тело, `.modal-footer`. Стили `.modal-box`: фон `var(--surface)`, граница `1.5px solid var(--border)`, скругление 16px, тень `var(--shadow-lg)`.

## 6. Главное окно и правый сайдбар страниц

**Эталон:** страница **Проекты**. Отступы и адаптив всех страниц (Дашборд, Аккаунты, Расписание, Профиль, Настройки системы) должны совпадать с ней. Правила совпадают с разделом редактирования проекта (`.project-view-body` / `.project-view-main` / `.project-view-sidebar`).

- **Обёртка:** `.page-body-wrap` — flex, основная колонка тянется, сайдбар справа.
- **Основная область:** `.page-main` — `flex: 1 1 0`, `min-width: 0`, скролл, отступы **50px** (≤768px — 16px, ≤639px — 12px), граница справа `1px solid var(--border)`, фон `var(--bg)`.
- **Правый сайдбар:** `.page-sidebar` — `flex: 0 0 25%`, `min-width: 240px`, отступы 24px (≤768px — 16px, ≤639px — 12px), фон `var(--surface)`. На всех страницах с двухколоночной раскладкой заполнять заголовком (`.page-sidebar-title.page-sidebar-title--large`) и подсказкой (`.page-sidebar-hint`) по образцу страницы Проекты.
- **Заголовок сайдбара:** `.page-sidebar-title` — шрифт `var(--text-lg)` (16px), начертание `var(--font-extrabold)`, цвет `var(--text)`, отступ снизу 12px. Вариант крупнее: `.page-sidebar-title--large` — размер `var(--text-xl)` (18px). Строка «заголовок + кнопка»: обёртка `.page-sidebar-header` — `display: flex`, `align-items: center`, `justify-content: space-between`, `gap: 12px`, `margin-bottom: 12px`; заголовок по левому краю, кнопка по правому; у заголовка внутри header отступ снизу сбрасывается.
- **Подсказки в сайдбаре:** `.page-sidebar-hint` — `var(--text-sm2)`, цвет `var(--text2)`, `line-height: 1.5`, без отступов.
- **Адаптив:** ≤1100px сайдбар 28%, ≤1000px — 30%; ≤768px — колонка: правый сайдбар **остаётся видимым**, переходит сверху (`order: -1`), основная область снизу; не скрывается и не сворачивается. Отступы 16px, у основной области снимается `border-right`; ≤639px — отступы 12px.

Менять пропорции, отступы и поведение на брейкпоинтах только вместе с соответствующими правилами для окна проекта (главное окно и раздел редактирования проекта должны вести себя одинаково).

## 7. Левый навигационный сайдбар

- **По умолчанию при ширине экрана < 950px** сайдбар в закрытом состоянии (класс `sidebar-closed` на `#app-shell`). Открывается только по нажатию кнопки (гамбургер). Порог задаётся в `app.js` (`SIDEBAR_AUTO_CLOSE_WIDTH`).
- **Закрытое состояние:** ширина 64px, только иконки пунктов меню; текст пунктов скрыт (`opacity: 0`, `width: 0`, `overflow: hidden` у `.nav-item span`). Стили для `#app-shell.sidebar-closed` (`.sidebar-nav`, `.nav-item`, скрытие span, размер иконок) **должны применяться на всех разрешениях**, где сайдбар виден (640px и выше), а не только при `min-width: 1024px`, иначе на планшетах (портрет) и телефонах (альбом) текст будет обрезаться.
- **≤639px:** левый сайдбар скрыт (`display: none`), навигация в мобильном drawer.

При изменении брейкпоинтов или стилей закрытого сайдбара проверять отображение в диапазоне 640–1023px (закрытое состояние = только иконки, без обрезанного текста).

## 9. z-index

| z-index | Элемент | Назначение |
|---------|---------|------------|

Стили в `app.css` (`.project-view-utm-help`, `.project-view-utm-help-tooltip`). При добавлении аналогичных подсказок на других страницах — использовать тот же приём (позиция слева).
| 50 | Топбар | Шапка приложения |
